<prompt id="2" task="multi-product ratio offer support">
We now need to extend the functionality of the basketPricer to cover the below scenario.

I have added documentation on basketPricer to indicate that where an offer applies across multiple product names the cheapest product per application should be given.

As below this should be applied to give the maximum possible discount, so the large shampoos should be calculated as a set of 3 to allow for one large shampoo to be free, leaving 2 smalls and a medium where the small is then free.

Currently we are looping over all offers for a given item in a basket.

We may now want a separate step in the basketPricer function to loop through all applicable offers. Instead of doing this within the for each item of a basket loop just push the applicableOffers for an item to an applicableBasketOffers array of offers, building up what offers are valid for our basket, and then we'll loop through these in a separate subsequent for each step passing all items that exist in our basket that exist in the offer itemNames array to calculateOfferDiscount so we can now handle ratio offers across multiple different named products

```
The supermarket is keen that the discounts should be applied fairly, so the rule must be interpreted to calculate the maximum possible discount given the set of applicable chosen products. This is complicated by the fact that a user may buy any number of products from the set {X}. Example:

The Supermarket currently has the following offer:
Buy three, get the cheapest one for free with any of the following:
Shampoo (Large)
Shampoo (Medium)
Shampoo (Small)
Given that the basket contains:
Shampoo (Large) * 3
Shampoo (Medium) x 1
Shampoo (Small) x 2
Result should give:
sub-total: £17.0
discount: £5.5
total: £11.5
The customer has got 1 large and 1 small shampoo for free.
```

Finally update the prompts.txt file to document the prompts as per the documentation there
</prompt>
